## Directory Purposes

**`src/components`:**
- Purpose: Reusable, composable UI primitives and widgets.
- Contains: Generic presentation blocks (`ChartCard`, `KPICard`, `SectionContainer`) and heavier widgets (`DataTable`, `AnalyticalChart`, `FilterBar`).
- Key files: `src/components/DataTable.tsx`, `src/components/AnalyticalChart.tsx`, `src/components/FilterBar.tsx`.

**`src/sections`:**
- Purpose: Domain-focused dashboard sections built from reusable components.
- Contains: Section-level composition for executive summary, quality, risk, temporal, and operational views.
- Key files: `src/sections/ExecutiveSummarySection.tsx`, `src/sections/OverviewSection.tsx`, `src/sections/OperationalImpactSection.tsx`.

**`src/pages`:**
- Purpose: Top-level screen orchestration.
- Contains: Full page assembly and state branching (loading/error/empty/content).
- Key files: `src/pages/DashboardPage.tsx`.

**`src/data`:**
- Purpose: Data source adapters and ingestion.
- Contains: Workbook/JSON loading from root dataset file.
- Key files: `src/data/loadSinistros.ts`.

**`src/utils`:**
- Purpose: Pure business and support utilities.
- Contains: Normalization, analytics, formatters, CSV export, executive HTML report generation, privacy masking.
- Key files: `src/utils/analytics.ts`, `src/utils/normalization.ts`, `src/utils/report.ts`, `src/utils/csv.ts`, `src/utils/privacy.ts`.

**`src/types`:**
- Purpose: Shared domain contracts and type aliases.
- Contains: Raw and normalized dataset shapes, filter/chart/KPI/risk/quality interfaces.
- Key files: `src/types/sinistro.ts`.

**`src/constants`:**
- Purpose: Centralized static constants and UI metadata.
- Contains: Filter defaults/labels, chart colors, section navigation IDs, future access profile constants.
- Key files: `src/constants/filters.ts`, `src/constants/charts.ts`, `src/constants/navigation.ts`, `src/constants/access.ts`.

## Naming Conventions

**Files:**
- React components and sections: PascalCase (`src/components/FilterBar.tsx`, `src/sections/InsightPanel.tsx`).
- Utilities/constants/data/types: camelCase or domain noun lowercase (`src/utils/analytics.ts`, `src/constants/filters.ts`, `src/data/loadSinistros.ts`, `src/types/sinistro.ts`).
- Root config files: tool-standard names (`vite.config.ts`, `tailwind.config.js`, `tsconfig.json`).

**Directories:**
- Source domain folders use lowercase plural nouns (`src/components`, `src/sections`, `src/constants`, `src/utils`, `src/types`, `src/data`, `src/pages`).

## Where to Add New Code

**New Feature:**
- Primary orchestration code: `src/pages/` (new page-level composition) or `src/sections/` (new dashboard block).
- Shared reusable UI elements: `src/components/`.
- Domain calculations or transformations: `src/utils/` with contracts in `src/types/sinistro.ts`.
- Tests: Create co-located tests near implementation (`src/**/file.test.ts[x]`) since no centralized test directory exists.

**New Component/Module:**
- Stateless reusable UI: place in `src/components/` using PascalCase filename and typed props.
- Domain section module: place in `src/sections/` and register navigation metadata in `src/constants/navigation.ts` when section is navigable.
- New data-access adapter: place in `src/data/` and keep parsing/normalization in `src/utils/normalization.ts`.

**Utilities:**
- Shared helpers: `src/utils/`.
- Static labels/options/tokens: `src/constants/`.
- Shared type definitions for any new utility output/input: `src/types/`.
